0 · Honest Reality Check

No system can guarantee "Stock X will rise 4–10% today." Not hedge funds, not quants, not this dashboard. If such a system existed, its owner would quietly become the richest person alive — not sell it. Anyone claiming otherwise is selling a fantasy.

What professionals actually build — and what we're building — is a decision-support system that stacks probabilities in your favor by combining sentiment + technicals + news + momentum, then surfaces a shortlist of candidates worth your own review. You make the final call and manage risk. That is what makes swing traders profitable — not prediction, but edge + discipline.

Reframed goal: Give you a daily, data-driven edge — a shortlist of high-probability swing candidates with transparent reasoning — while enforcing risk management so winners outweigh losers over time.

4 · What the Dashboard Includes

📊Tab 1 · Market Pulse

Fear & Greed dial (0–100), Today's Bias card with confidence % + reasons, Nifty 50 & Bank Nifty snapshot, India VIX, and FII/DII institutional flows.

📰Tab 2 · News & Sentiment

Live scored headlines (pos/neu/neg), sentiment trend chart, filter by market/sector/stock, and an auto-generated "what's moving the market today" summary.

🔥Tab 3 · Sector Rotation

Heatmap of all NSE sectoral indices, ranked by % change + relative strength + momentum. Highlights hot sectors where money is flowing, with news-sentiment overlay.

🎯Tab 4 · Swing Candidates

Technical + sentiment screener. Per candidate: RSI, MACD, volume spike, breakout status, a transparent Swing Score (0–100), and suggested entry / stop-loss / target with risk:reward.

🛡️Tab 5 · Risk & Position Sizing

Position-size calculator (capital + risk %), stop/target calculator, portfolio risk overview, and a discipline checklist. This is what makes you profitable.

📈Tab 6 · Journal & Backtest

Log trades, track win rate & avg P/L. Stored daily signals let us backtest whether the Swing Score actually works — the professional feedback loop.

8 · MCP Setup — Quick Start

Detected: ✅ claude, node v26, npm, git · ❌ gh, docker (not needed). Full guide: docs/mcp-setup-guide.md.

1 · GitHub MCP ⭐

Create a fine-grained PAT at github.com/settings/tokens (Contents, PRs, Issues: R/W), then:

claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  -H "Authorization: Bearer github_pat_YOUR_TOKEN_HERE" \
  -s user

2 · Fetch MCP ⭐ (news / NSE during dev)

claude mcp add fetch -s user -- npx -y @modelcontextprotocol/server-fetch

3 · Filesystem MCP (optional extra folders)

claude mcp add filesystem -s local \
  -- npx -y @modelcontextprotocol/server-filesystem \
     /Users/alok.ranjan/Documents/Projects/indian-market-dashboard

4 · Playwright MCP (optional scraping)

claude mcp add playwright -s user -- npx -y @playwright/mcp@latest

5 · Figma MCP (optional design)

claude mcp add figma -s user \
  -e FIGMA_API_KEY=figd_YOUR_TOKEN \
  -- npx -y figma-developer-mcp --stdio

Verify with claude mcp list · remove with claude mcp remove <name>.

Minimal to start: just add GitHub + Fetch now; add the rest when needed.
